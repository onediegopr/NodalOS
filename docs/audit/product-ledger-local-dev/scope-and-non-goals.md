# Scope And Non-Goals

## In Scope

- Product Ledger local/dev evidence line.
- Current authority docs and guard tests from E2 through E5.
- Manual/discovery-only validation commands.
- Historical entrypoint interpretation and stale-doc cross-link quality.
- Blocked-state clarity for public/product, Production route, latest pointer, read precedence and product authority.

## Out Of Scope

- Runtime/product implementation.
- Public/product route activation.
- Production route activation.
- Latest pointer creation or overwrite.
- Active read precedence.
- Product authority or product read-model authority.
- Productive DI/service registration.
- Productive command handlers.
- UI product actions.
- DB/migration.
- Provider/cloud/network.
- KMS/WORM/external durable trust.
- Command/shell/subprocess execution enablement.
- Browser/CDP/WCU/OCR/Recipes live automation.
- CI enforcement.
- Release/commercial/public readiness.
- External API calls or external audit submission.

## Non-Inference Rule

Passing tests or collecting local/dev evidence does not mean Product Ledger is product-ready. The packet is for read-only review only.
