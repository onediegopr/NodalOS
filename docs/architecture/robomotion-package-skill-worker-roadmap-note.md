# Robomotion Package Skill Worker Roadmap Note RM-LITE-001

## What Robomotion Represents

Robomotion is a useful reference for package, skill, worker and connector lifecycle thinking in agentic automation.

Useful benchmark areas:

- packages
- skills
- registry
- workers
- connectors
- package lifecycle
- worker boundaries

## TAKE FUTURE

- future Package / Skill Manifest
- future Internal Skill Registry
- future Worker Boundary Contract
- future connector lifecycle model
- future package versioning
- future skill provenance

## ADAPT FUTURE

- packages remain Core-governed
- registry remains internal, not a public marketplace
- worker boundaries include policy and evidence requirements
- workers do not become the brain
- skills carry approval and risk metadata

## REJECT NOW

- full cloud-native RPA platform
- heavy multi-worker runtime
- public marketplace
- sensitive cloud runtime
- massive connector surface
- external Robomotion dependency
- bot bypassing
- aggressive scraping

## Future Roadmap

- RM-001 Package / Skill Manifest V1
- RM-002 Internal Skill Registry V1
- RM-003 Worker Boundary Contract

## Non-Goal

This block does not implement packages, registries, workers or connectors. It records roadmap intent only.
