# OpenRPA Inspired Patterns: No Code Copy

## Inspiration Only

This line borrows high-level workflow ideas from OpenRPA/OpenCore:

- workitems with payload/files/priority/next-run metadata,
- processing/success/retry/failed state transitions,
- business failure vs application failure,
- retry policy with next-run metadata,
- prepare/process/verify/cleanup lifecycle,
- human handoff for intervention.

## Explicit Non-Use

- OpenRPA dependency: NO.
- OpenRPA code copied: NO.
- XAML imported: NO.
- Runtime host imported: NO.
- Browser extension/native messaging added: NO.
- File watcher/scheduler/recorder/replay added: NO.
- Real browser automation added: NO.
- Real desktop/computer-use automation added: NO.

## NODAL OS Improvements

NODAL OS keeps the model local-first, evidence-by-reference, timeline-compatible, redaction-compatible, approval-compatible, and policy-gated. Live adapters are future-gated and absent from this block.
